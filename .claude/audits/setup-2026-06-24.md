```json
{
  "permissions": {
    "allow": ["Bash:**", "Read:**", "Write:**", "Edit:**", "Glob:**", "Grep:**"],
    "deny": ["Bash:rm -rf /*", "Bash:curl *", ...]
  },
  "model": "sonnet",
  "hooks": {
    "afterEdit": [
      {
        "description": "Verify C# code builds after edits",
        "command": "dotnet build --nologo --verbosity quiet",
        "when": {"glob": "scripts/**/*.cs"}
      }
    ]
  },
  "environment": {
    "DOTNET_CLI_TELEMETRY_OPTOUT": "1",
    "DOTNET_SKIP_FIRST_TIME_EXPERIENCE": "1"
  }
}
```

### Key Development Standards

**Spec-Driven Development**:
- Never modify source code without a specification
- Specifications are: failing tests, GitHub issues, or design docs
- Follow Red-Green-Refactor cycle

**Git Conventions**:
- Conventional Commits format mandatory
- Types: feat, fix, docs, style, refactor, perf, test, build, ci, chore
- Always include `Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>`

**Quality Standards**:
- 80% minimum code coverage
- Zero warnings policy (all warnings as errors)
- Nullable reference types enabled
- Modern C# patterns (file-scoped namespaces, etc.)

**Godot Conventions**:
- Separate testable logic from Godot-specific rendering
- Cache GetNode() calls in _Ready()
- Use signals for decoupled communication
- One class per file, organized by feature
